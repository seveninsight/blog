用Hexo搭建博客（1）：在本地创建并预览博客

发表于 2018-12-30 | 更新于 2019-01-03 | 分类于 搭建博客 ， Hexo

Hexo 是一个快速、简洁且高效的博客框架。hexo 出自台湾大学生tommy351之手，是一个基于Node.js的静态博客程序，其编译上百篇文字只需要几秒。Hexo 生成的静态网页可以直接放到 GitHub Pages 等平台上。

1 安装 Hexo 的前提

在安装 Hexo 前，必须先保证电脑中已安装下列应用程序：

  • Node.js
  • Git

2 安装 Git

Git的客户端很多，官方的是msysgit（也就是 Git for Windows）。

绿色版本的Git：
下载地址：https://github.com/git-for-windows/git/releases
下载下来设置一下环境变量即可，Git_HOME，%Git_HOME%\bin之类的。

3 安装 NodeJS

  1. 下载 Node.js 安装程序

    官方网站：https://nodejs.org/en/

  2. 安装 Node.js

    windows系统建议使用安装程序进行安装。安装时，请勾选Add to PATH选项。
    按照提示一步一步安装即可。

  3. 测试 Node.js 是否安装成功

    输入node -v命令查看nodejs版本。

4 配置淘宝NPM镜像（可选）

cnpmjs.org 是一个非常棒的 npm 国内镜像。由于其使用量越来越大，加上淘宝内部也有很多项目使用 NodeJS，于是，淘宝正式基于 cnpmjs 推出了镜像服务。

淘宝NPM镜像 是一个完整 npmjs.org 镜像，可以用此代替官方版本(只读)，同步频率目前为 10分钟 一次以保证尽量与官方服务同步。

  • 使用说明

可以使用淘宝NPM镜像定制的cnpm(gzip 压缩支持) 命令行工具代替默认的npm：

  • 安装模块

从 registry.npm.taobao.org 安装所有模块. 当安装的时候发现安装的模块还没有同步过来, 淘宝 NPM 会自动在后台进行同步, 并且会让你从官方 NPM registry.npmjs.org 进行安装. 下次你再安装这个模块的时候, 就会直接从 淘宝 NPM 安装了.

  • 同步模块

直接通过 sync 命令马上同步一个模块, 只有 cnpm 命令行才有此功能.

可以直接通过 web 方式来同步: /sync/connect

5 安装 Hexo

确认电脑已经安装 Git 和 Node.js 必备程序后，接下来只需要使用 npm 即可完成 Hexo 的安装。

因为安装了淘宝的NPM，所以可以使用cnpm命令：cnpm install -g hexo-cli

  4. Hexo 博客框架的目录结构

    新建完成后，指定文件夹的目录如下：

    .
├── node_modules/       # 依赖包
├── scaffolds/          # layout模版 文件夹。
|                       # 当新建文章时，Hexo 会根据 scaffold 来建立文件。其中的md文件可以添加编辑
├── source/             # 资源文件夹是存放用户资源的地方。
|   |                   # 除 _posts 文件夹之外，开头命名为 _ 的文件、文件夹和隐藏的文件将会被忽略。
|   |                   # 该页面对应repo的根目录，404文件、favicon.ico文件，CNAME文件等都应该放这里
|   |                   # Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。
|   ├── _posts/         # 发布文章
|   └── _drafts/        # 草稿文章
|
├── themes/             # 主题文件夹。Hexo 会根据主题来生成静态页面。
├── .gitignore
├── _config.yml         # 全局配置文件
├── package.json        # 应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，您可以自由移除。
└── package-lock.json

    这是初始化还没有做其他操作时的目录结构，之后，除此以外还会有db.json、public、scripts、.deploy。
    其中，public存放由source文件夹解析生成的静态文件。
