用Hexo搭建博客（2）：把博客部署到 github

发表于 2019-01-01 | 更新于 2019-01-03 | 分类于 搭建博客 ， Hexo

用 Hexo 创建好本地博客，然后使用hexo server命令，在浏览器中输入http://localhost:4000/预览，如果博客的内容与效果都感觉棒棒的，就可以开始下一步 —— 部署到 github 啦。

1 在 github 上创建仓库

要将博客部署到 Github Pages，首先得创建一个存放博客文件的仓库。

Github Pages 博客要求仓库名称必须是固定格式的：

github用户名.github.io

例如，我的 github 用户名为seveninsight，所以仓库名称必须是seveninsight.github.io。

不知道怎么创建 github 仓库的同学，先去百度一下吧。

仓库创建好以后，点击进入这个仓库，点击【Settings】。

把页面往下拉，然后会看到【GitHub Pages】那一栏，如下图，方框中的就是部署博客的URL。

2.2 配置远程 git 仓库路径

因为要提交到 github 上的远程 git 仓库，所以要配置git仓库的路径。

远程 git 仓库的路径，可以在 github 上查看。
进入 git 仓库，点击右侧的绿色按钮【Clone or download】，如下图，下拉框会弹出URL，点击右侧小图标复制。

打开 Hexo 的配置文件，找到【deploy】配置项的repo进行配置，粘贴刚刚复制的 git仓库路径即可。

3 配置 git

3.1 关联到远程仓库

按照上一节讲的操作，在第4小节中，我们执行hexo deploy命令时，实际上会自动关联到远程仓库，可以不用手动再配置。

之所以把这一步单独提出来，是因为要特别注意，如果不配置，用户和邮箱会默认用 git 的全局配置。

3.2 配置不忽略大小写，解决 404 问题

因为 git 提交文件时，默认配置是忽略大小写的。这样会导致一个问题，如果博客的标签或者分类涉及英文字母的大小写，有时提交到远程 git 仓库后，大小写会出现变化，导致访问页面失败，出现404。所以要配置不忽略大小写。

不忽略大小写的配置文件属于 git 的配置，在根目录下的文件夹下.deploy_git，有一个隐藏的文件夹.git（如果看不到，要设置【显示隐藏的文件、文件夹和驱动器】，不会的百度一下），配置文件叫做config，把ignorecase配置成 false 即可。

4 部署到 github

终于要部署到 github 了，是不是很激动。别着急，在部署之前，要先生成静态的HTML文件。

4.1 生成静态 HTML 文件

进入博客的根目录，打开命令行CMD，输入命令hexo generate，这样就会生成静态文件了。
生成的文件放在博客根目录的public文件夹下，我们部署到 GitHub Pages ，其实就是把这个文件夹里面的文件复制一份到.deploy_git，然后提交到远程 git 仓库。

5 同时部署到多个 git 代码托管平台

5.1 常见的 git 代码托管平台

除了国外的 github，国内的 git 代码托管平台有：

  • 码云：https://gitee.com 码云是开源中国（oschina）推出的基于 Git 的代码托管服务
  • https://coding.net/

github 访问速度比较慢，且百度不能搜索到 GitHub Pages 上的博客，建议同时把博客部署在国内的 git 代码托管平台上。

接下来以码云为例，讲解一下怎么同时部署到多个平台。

5.2 注册码云账号，创建远程仓库

具体步骤与 github 类似，不再赘述。

这里只强调2点：

第一：

因为 Github Pages 对存放博客的 git 仓库名称有要求。为了方便以后统一部署，所以其他平台上存放博客的仓库取名应该与 github 上的仓库名称相同。

比如我就在码云上创建了一个仓库，命名与github一样，也叫seveninsight.github.io。

第二：

码云上创建的仓库，不能直接托管静态博客，需要开启Gitee Pages服务才可以，如下图，点击进入 Gitee Pages 页面，然后点击开启。

5.3 相关的配置

5.3.1 在 Hexo 的配置文件中添加远程仓库路径

1）与 github 类似，先获取代码仓库的路径：

https://gitee.com/seveninsight/seveninsight.github.io.git

2）打开 Hexo 根目录下的配置文件_config.yml，配置仓库路径

5.3.2 配置不同 git 仓库的账号密码

1）首先添加环境变量

变量名：HOME
变量值：%USERPROFILE%

2）编辑配置文件

在 C:\Users\【计算机用户名称】目录下，新建名称为_netrc的文本文件。

文件内容格式如下：

machine github.com  #布署网站名
login 【github的账号名称】
password 【github的账号密码】

machine gitee.com
login 【码云的账号名称】
password 【码云的账号密码】

到这里就全部配置好了，接下来只要执行命令hexo d，就可以一键部署到多个平台了。
