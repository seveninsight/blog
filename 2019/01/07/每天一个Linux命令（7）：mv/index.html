每天一个Linux命令（7）：mv

发表于 2019-01-07 | 分类于 Linux ， 每天一个Linux命令

mv命令是move的缩写，用来对文件或目录重新命名，或者将文件从一个目录移到另一个目录中，是Linux系统下常用的命令。

1 功能

mv命令用来对文件或目录重新命名，或者将文件从一个目录移到另一个目录中。

2 命令格式

mv [选项] 源文件 目标文件

【选项说明】

--backup=<备份模式>：若需覆盖文件，则覆盖前先行备份；
-b：当文件存在时，覆盖前，为其创建一个备份；与 --backup 类似，但是不接受参数
-f：若目标文件或目录与现有的文件或目录重复，则直接覆盖现有的文件或目录；（默认选项）
-i：交互式操作，覆盖前先行询问用户，如果源文件与目标文件或目标目录中的文件同名，则询问用户是否覆盖目标文件。用户输入”y”，表示将覆盖目标文件；输入”n”，表示取消对源文件的移动。这样可以避免误将文件覆盖。
--strip-trailing-slashes：删除源文件中的斜杠“/”；
-S<后缀>：为备份文件指定后缀，而不使用默认的后缀；
--target-directory=<目录>：指定源文件要移动到目标目录；
-u：当源文件比目标文件新或者目标文件不存在时，才执行移动操作。
-v: 解释执行了哪些操作

【参数说明】

1）当第2个参数即目标文件存在时：

  • 如果目标文件是一个目录，将源文件移到目标目录中，此时参数1即源文件可以有多个；
  • 如果目标文件是一个文件，将源文件重命名为目标文件名，并覆盖目标文件，此时参数1只能有一个；

2）当第2个参数即目标文件不存在时：

  • 将源文件重命名为目标文件名，并覆盖目标文件，此时参数1只能有一个；

3、使用-b选项，文件覆盖前做备份

说明：

  • -b不接受参数，mv会去读取环境变量VERSION_CONTROL来作为备份策略。
  • --backup该选项指定如果目标文件存在时的动作，共有四种备份策略：
    1. CONTROL=none或off : 不备份。
    2. CONTROL=numbered或t：数字编号的备份
    3. CONTROL=existing或nil：如果存在以数字编号的备份，则继续编号备份m+1…n：执行mv操作前已存在以数字编号的文件log2.txt.~1~，那么再次执行将产生log2.txt~2~，以次类推。如果之前没有以数字编号的文件，则使用下面讲到的简单备份。
    4. CONTROL=simple或never：使用简单备份：在被覆盖前进行了简单备份，简单备份只能有一份，再次被覆盖时，简单备份也会被覆盖。

4、使用-i选项，覆盖同名文件前询问

5、使用-f选项，覆盖同名文件前不询问，默认是-f选项。
